LESSON 5 — FILE OWNERSHIP:
    run "ls -l"
    output:-rw-r--r-- or drwxrwxrwx  or -rwxrwxrwx
    
    BREAKDOWN:
        FIRST CHARACTER(-):means files but if it was (d) means Directory/Folder
        NEXT 3 CHARACTERS(rw-) or (rwx): permissions for owners meaning read,write,no execute
        NEXT 3 CHARACTERS(r--): Permissions for GROUP 
        Last 3 CHARACTERS(r--): Permissions for EVERYONE ELSE

    3 MAIN PERMISSIONS:
        r=read
        w=write
        x=execute

        REAL EXAMPLE: -rwxr-xr--(file)
            UserType                Permissions
            -Owner                  read,write,execute                
            -Group                  read,execute
            -Others                 read
        
        WHY THIS MATTERS IN AWS:
            Deployment problems often happen because:
                - Django cannot access files
                - Nginx lacks permission
                - SSH key permissions wrong
                - Gunicorn socket inaccessible

            Linux permissions are everywhere.

LESSON 12 — IP ADDRESSES

Every device on a network has an address which shows their identify just like a home address and a phone number,without an IPAddress computers can't find each other

        Types of IP Address:
            -Private IP Adress:An Address not accessible from the public internet, used in home,offices and private networks e.g 192.168.x.x, 10.x.x.x, 172.x.x.x
            -Public IP Address: An Address that can be Accessible on globally on the public internet
            e.g AWS,EC2 Servers Have Public IP's

        VERY IMPORTANT: Your laptop usually has:
            - private IP internally
            - router exposes public IP externally

LESSON 13 — PORTS,This is MASSIVE.A commununication door into a computer

    ANALOGY:Think:
        - IP Address = Apartment Building
        - Port = Apartment Number

        Example: 192.168.1.5:8000
            -Means: 
                - Go to computer 192.168.1.5
                - Enter through door 8000

        COMMON PORTS:

        | Port | Purpose           |
        | ---- | ----------------- |
        | 80   | HTTP              |
        | 443  | HTTPS             |
        | 22   | SSH               |
        | 5432 | PostgreSQL        |
        | 3306 | MySQL             |
        | 6379 | Redis             |
        | 8000 | Django dev server |

        IMPORTANT: One computer can run MANY services because each service uses different ports.
        Example:
            Nginx → Port 80
            Django → Port 8000
            PostgreSQL → Port 5432
            Redis → Port 6379


LESSON 14 — WHAT HAPPENS WHEN YOU VISIT A WEBSITE: Suppose you visit: google.com
Here’s what REALLY happens:

    STEP 1 — 
        DNS LOOKUP:Your browser asks: “What is google.com’s IP address?”
        DNS responds: 142.250.xxx.xxx
    
    STEP 2 — CONNECT TO SERVER: 
        Browser connects to: 142.250.xxx.xxx:443
        Port 443 = HTTPS.
    STEP 3 — REQUEST SENT:Browser sends:GET/
        Meaning:“Give me the homepage.”

    STEP 4 — SERVER RESPONDS
        Server sends:
            - HTML
            - CSS
            - JavaScript
            - images
        Back to browser.
        BOOM: That’s the web.

LESSON 21 — WHAT IS CORS?

    You’ve probably seen CORS errors before.Now let’s understand WHY.

    Suppose:
        - Frontend:localhost:3000
        - Backend:localhost:8000

    These are DIFFERENT origins.

    Browser security says: “Can frontend safely access backend?”
    If backend doesn’t allow it:
    → CORS error.
    e.g django-cors-headers
