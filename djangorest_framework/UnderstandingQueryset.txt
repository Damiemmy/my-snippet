# 🧠 Understanding `get_queryset()` in RBAC (Django REST Framework)

## 🔍 The Question

Does this line:

```python
return Product.objects.all()
```

mean that an **authenticated user who is not a host/admin/staff** will see all products?

👉 **Yes — exactly.**

---

# 🧠 Step-by-Step Execution Flow

```python
def get_queryset(self):
    user = self.request.user

    if not user.is_authenticated:
        return Product.objects.all()

    if user.is_superuser:
        return Product.objects.all()

    if user.role == 'admin':
        return Product.objects.all()

    if user.is_staff:
        return Product.objects.all()

    if user.role == 'host':
        return Product.objects.filter(user=user)

    return Product.objects.all()
```

# ⚠️ Important Design Question

> Should normal users see ALL products?

### ✅ In most apps (like Airbnb):

Yes — listings are public.

---

### 🔒 But in stricter systems:

You might want:

```python
return Product.objects.filter(is_published=True)
```

👉 Only approved/public listings are visible

---

# 🔥 Cleaner, More Explicit Version

```python
def get_queryset(self):
    user = self.request.user

    # Public users
    if not user.is_authenticated:
        return Product.objects.all()

    # High-level access
    if user.is_superuser or user.role == 'admin' or user.is_staff:
        return Product.objects.all()

    # Host → own products
    if user.role == 'host':
        return Product.objects.filter(user=user)

    # Normal user
    if user.role == 'user':
        return Product.objects.all()

    return Product.objects.all()
```

---

# 🧠 Why This Version is Better

* ✅ No hidden logic
* ✅ Easier to read
* ✅ Easier to debug
* ✅ Explicit role handling

---

# 🛡️ Important Note

That last line:

```python
return Product.objects.all()
```

👉 Acts as a **safety net**

Without it:

* Your function may return nothing ❌
* Your API can break ❌

---

# 🧠 One-Line Truth

> The final return handles authenticated users who are not host/admin/staff — meaning normal users.
