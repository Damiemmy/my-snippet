DEPLOYEMENT FOR NEXT.JS APP(WEB SERVICE):
    package.json file must have :
    {
        "scripts": {
            "dev": "next dev",
            "build": "next build",
            "start": "next start -p $PORT"
        }
     }

    {
        "engines": {
            "node": ">=18"
        }
    }

    BUILD COMMAND FOR NEXT.JS ON RENDER: npm install; npm run build
    START COMMAND FOR NEXT.JS ON RENDER: npm install; npm run start
    AND FINALLY:
        NODE_ENV=production
        NEXT_PUBLIC_API_URL=https://your-backend.onrender.com

DEPLOYMENT FOR REACT(VITEAPP STATICSITE):
    package.json file must have :
        {
            "scripts": {
                "dev": "vite",
                "build": "vite build",
                "preview": "vite preview"
            }
        }
    

        | Setting           | Value                          |
    | ----------------- | ------------------------------ |
    | Build Command     | `npm install && npm run build` |
    | Publish Directory | `dist`                         |


    üîê Environment Variables in Vite (VERY IMPORTANT):

        Vite only exposes env vars that start with VITE_

        VITE_API_URL=https://your-backend.onrender.com

        Usage in code:
        const apiUrl = import.meta.env.VITE_API_URL;
        ‚ùå process.env will NOT work in Vite.


SUMMARY BETWEEN VITE AND NEXT.JS

    | Feature             | Next.js      | Vite                 |
| ------------------- | ------------ | -------------------- |
| SSR / API routes    | ‚úÖ Yes        | ‚ùå No (frontend only) |
| Build output        | `.next/`     | `dist/`              |
| Start server        | `next start` | Static files         |
| Best Render service | Web Service  | **Static Site**      |
| Complexity          | Medium‚ÄìHigh  | **Low**              |

| Framework   | Env Access        | Where it Runs    |
| ----------- | ----------------- | ---------------- |
| **Next.js** | `process.env`     | Server + Browser |
| **Vite**    | `import.meta.env` | Browser only     |

CONFIGURE THIS ON RENDER(FRONTEND):
NEXT_PUBLIC_BASE_URL:https://backend
